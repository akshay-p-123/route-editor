Do NOT re-derive `stop_id in original_ids and not stop_id.startswith("custom_")` anywhere else in the endpoint; every "is this an existing stop?" decision reads `classifications[i] == "existing"`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/BFF → FastAPI | Authenticated request body (`original_stops`, `proposed_stops`, `trip_id`) crosses into the backend; stop_ids may be forwarded into the MTD API URL path |
| FastAPI → MTD API v3 | `stop_id` becomes a path segment in `/stops/{stop_id}/departures` |
| FastAPI → OSRM | User-influenced lat/lon become query coordinates (hardcoded host) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering/Injection | stop_id forwarded to MTD `/stops/{stop_id}/departures` | mitigate | Apply `_STOP_ID_RE.match()` to every stop_id before passing to `_get_delays_for_stops` (identical to `/trip-updates` lines 1032-1034) |
| T-05-02 | Spoofing (unauth use) | `POST /estimate-travel-time` | mitigate | `Depends(_user_id)` Supabase JWT validation, same as `/trip-updates` and `/trip-modifications/import` |
| T-05-03 | Denial of Service | Large `proposed_stops`/`original_stops` causing OSRM/MTD fan-out | accept | `_osrm_route` already caps at 12 sampled waypoints via `_sample_stops`; realistic bus routes are bounded (~20-40 stops). Low value target, request-body only — accept without an explicit cap for MVP |
| T-05-04 | Spoofing (SSRF) | OSRM coordinates | accept | OSRM host (`router.project-osrm.org`) is hardcoded; only lat/lon params are user-influenced — same surface as already-shipped `/export/png` and GTFS export |
</threat_model>
