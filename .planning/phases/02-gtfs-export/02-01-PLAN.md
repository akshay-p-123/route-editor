<objective>
Build the backend GTFS static export endpoint: `GET /api/gtfs/export/{reroute_id}` fetches a reroute's saved routes from Supabase, builds all 8 GTFS DataFrames, writes them to a zip via gtfs-kit, and returns the zip bytes. This is the thinnest end-to-end backend slice of the user-facing capability "download a spec-compliant GTFS zip for a reroute package."

Purpose: This endpoint is the engine behind the entire phase. Once it returns a valid zip, the frontend slice (02-02) only has to wire a button to it, and validator compliance (02-03) only has to confirm it.
Output: Extended `backend/app/routers/gtfs.py` with the export endpoint and DataFrame builders; bootstrapped pytest test suite under `backend/tests/`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → FastAPI export endpoint | Untrusted Bearer token + reroute_id path param cross here |
| FastAPI → OSRM public server | Outbound coordinates (from DB, not request body) cross here |
| FastAPI → gtfs-kit/tempfile | Server-side DataFrame → zip on local disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing/EoP (IDOR) | export_gtfs reroute fetch | mitigate | Ownership check `.eq("id", reroute_id).eq("user_id", user_id)` before any data fetch; 404 on miss. Covered by test_export_ownership. |
| T-02-02 | Denial of Service | gtfs-kit zip build on malformed route data | mitigate | Coordinates originate from Supabase (validated at save time), not request body; reroute_id is UUID-typed (FastAPI 422 on malformed); single reroute per request bounds memory. |
| T-02-03 | Information disclosure / availability | OSRM outbound call | mitigate | OSRM failure caught, logged as warning, falls back to straight-line geometry + 60s/stop timing (D-06); export never fails on OSRM unavailability. Exercised by test_export_endpoint with _osrm_route monkeypatched to None. |
| T-02-SC | Tampering | npm/pip installs | mitigate | No new packages beyond pytest/pytest-asyncio (dev-only, PyPI-standard). gtfs-kit already approved in Phase 1 audit. See RESEARCH §Package Legitimacy Audit. |
</threat_model>
