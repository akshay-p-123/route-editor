<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/BFF → FastAPI (`/api/gtfs/trip-updates`) | Untrusted `stop_ids` query string and Authorization header cross here |
| FastAPI → MTD API v3 (`api.mtd.dev/stops/{id}/departures`) | Server-side egress with secret `X-ApiKey`; stop_id becomes a URL path segment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `stop_ids` query param → `f"/stops/{stop_id}/departures"` path segment in `get_stop_departures` | mitigate | Each stop_id is interpolated into an httpx request path; httpx URL-encodes path components (e.g. `/`, `?`, `#` → `%2F`/`%3F`/`%23`), so a crafted stop_id cannot break out of the `/stops/{id}/departures` path or inject query params. Endpoint additionally splits on comma, strips whitespace, and drops empties before use. No SQL/shell context is reached — stop_ids never touch a database or subprocess. |
| T-03-02 | Spoofing | `GET /trip-updates` auth | mitigate | Endpoint declares `user_id: str = Depends(_user_id)` (D-08). `_user_id` calls Supabase `sb.auth.get_user(token)` which validates JWT signature + expiry on every request and raises `HTTPException(401)` on failure. Verified by `test_unauthenticated_401`. |
| T-03-03 | Information Disclosure | MTD `X-ApiKey` secret | mitigate | Key stays server-side: set only in `services/mtd._headers()` via `settings.mtd_api_key`; never returned in the `dict[str, int]` response. BFF rule keeps the browser from reaching FastAPI directly. |
| T-03-04 | Denial of Service | Endpoint spammed → MTD rate-limit / fan-out amplification | mitigate | 60s `_dep_cache` TTL (D-03) absorbs repeated identical stop-set requests, collapsing them to one MTD fan-out per minute. Order-independent cache key (`sorted(ids)`) maximizes hit rate. Upstream MTD 429 surfaces via `_get`'s `raise_for_status()` and is caught per-stop by `asyncio.gather(return_exceptions=True)` → logged + omitted, never crashing the request. Residual risk (unbounded unique stop-sets) accepted for MVP per RESEARCH Pitfall 5 — demand-filled, rare in the single-reroute estimation workflow. |
| T-03-05 | Denial of Service | Per-stop MTD failure cascading to full-request failure | mitigate | `return_exceptions=True` + `isinstance(result, Exception)` omit-and-continue (warn-don't-crash); partial results returned. Verified by `test_per_stop_error_omitted`. |
| T-03-SC | Tampering | npm/pip installs | accept | No new packages installed this phase (RESEARCH Package Legitimacy Audit: httpx/fastapi/pydantic pre-existing, verified Phase 1/2). No install task exists; legitimacy checkpoint not required. |
</threat_model>
